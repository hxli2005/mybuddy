# 小布桌宠 · 最终产品形态(v1 定稿)

> 本文档是产品形态的唯一真相源,回答"她是什么";工程规格(怎么建)从本文派生,交 Codex 执行。
> 形态与宿主解耦:无论最终用 VPet.Core 还是自写动画播放器,本文一个字不改。
> 前序:docs/VPET_PHASE2_SPEC.md(桥接工程规格,四轮评审)+ 本会话 Fable 反方轮裁决。

## 0. 定义与反定义

**一句话**:小布桌宠是一个住在你桌面角落的**同居者**——身体会饿会困会蹭你,记忆里存着你们的共同生活,并且知道什么时候该说话、什么时候只是在。

**它是**:一个有身体、有作息、有记忆、懂分寸的常驻角色。
**它不是**:助手(不以效率为身份)、电子宠物游戏(不用数值勒索照顾)、聊天窗(对话是生活的一部分,不是全部)。

**北极星(私人罗盘,不进论文)**:第 14 天她还在桌面,且当天有一次不是你先开口的互动。

## 1. 体验宪法(裁决未来一切争论的八条)

1. **一个灵魂,一个声音**:身体发出的每一个字都出自小布(引擎生成或引擎审过的模板)。身体永远不自行说话。
2. **智能全在引擎,身体只表达与传感**:壳里没有策略,只有渲染和传感器。所有"该不该、什么时候、说什么"归大脑。
3. **沉默是默认态**:说话需要理由(被摸、被喂、你回来了、到点了、你先开口)。没理由时,她在做自己的事。
4. **打扰的代价高于错过**:主动触达受在场/冷却/静默时段/每日预算约束;错过一次问候无所谓,吵到一次工作是事故。
5. **照顾是被邀请的,不是被勒索的**:她会饿会蔫,但不会病死、不倒计时、不发愧疚通知。你照顾她因为想,不因为怕。
6. **关系用行为表达,不用数值表达**:UI 里没有好感度条、没有等级、没有成就。亲疏体现在她的姿态、语气、靠近的距离。数字只活在遥测里供论文用。
7. **记忆必须回到生活里**:共同经历(投喂/摸头/共处/聊过的事)会在之后的日子里被她自然提起;记忆的管理界面永远留在 Web 机房,不进桌宠。
8. **一切行为可审计**:每次互动、每次主动、每次沉默的理由都落遥测——她既是产品,也是论文的实验仪器。

## 2. 她的一天(形态即行为,六拍)

| 拍 | 行为 | 背后系统 |
|---|---|---|
| 1 | 09:12 你坐下,她从打盹里起来:"来了?今天先弄周报吧" | 在场感知 + user_back + 记忆检索 |
| 2 | 上午她在屏幕角落自己看书/伏案/望窗外,不出声 | living_state → idle 行为(大脑下发) |
| 3 | 11:40 你摸头两下:眯眼蹭一下(反射);当天第一次那下,她说了句"嗯,在呢。" | 触摸反射层 + 升格预算(≤28 字,不反问不建议) |
| 4 | 14:00 她饿了,蔫蔫的但不吵你;你从食盘拖了份咖喱给她;晚上聊天她自己提起"下午那份咖喱" | 生理曲线 + 投喂交互 + shared_moment 回流 |
| 5 | 18:30 你离开;21:00 回来,她只说一句:"你不在的时候我攒了两件事:一个提醒,还有一次想叫你歇会儿。" | 在场门控 + digest/overdue |
| 6 | 00:30 你还在肝论文,她也困了;你说晚安,她去睡 | 作息曲线一致 + 晚安仪式 |

**她的一周(周检三项)**:①至少一次记忆回流被你接住(你回应了她提起的往事);②至少三次共处 session;③夜间打扰 = 0。

**过滤门(不变)**:任何新想法先答"它出现在这一天/这一周的哪一拍?"答不出,不做。

## 3. 能力全景(六层,终态)

### 3.1 身份层
- **声音统一 by construction**:自建壳内没有第二人格的台词系统;低状态哼唧、睡话、自言自语全部由引擎按 persona 生成(低频、可关)。
- **身体自认知**:persona 里写明她的身体样貌与能力边界,她知道自己"住在桌面右下角、会被摸头、不会走出屏幕"。
- 一身一魂:无多存档、无皮肤切换人格。

### 3.2 身体层(生理)
- **生理真相源在大脑**(本轮关键决策):饥饿/精力/心情三条曲线由引擎拥有和演化(时间衰减 + 互动影响 + 作息),壳只渲染状态和提供交互面。VPet 游戏层的数值系统不进终态——一致性护栏问题就此 by construction 消失,`_body_state_conflicts` 降级为回归守卫。
- **曲线原则**:简单、可读、无惩罚。饿→蔫(视觉),不病死;困→打盹;心情低→安静黏人。
- **触摸**:头/身两区;反射动画永远即时本地;有声反应走升格预算。
- **投喂**:小食盘 3–5 样固定食物,拖给她吃;无货币、无商店、无消耗焦虑。每次投喂是 shared_moment 候选。
- **作息**:她有睡眠时段(与 quiet_hours 一致);深夜互动她会困着回应;吵醒有台词。

### 3.3 时机层
- **注意力感知触达(终态完整版)**:引擎基于在场信号(输入空闲/回场/久坐时长/全屏与会议检测)决定触达时机;所有主动受冷却 + 每日预算 + "今天安静"开关约束。
- **回场摘要**:digest 一句话 + overdue 持久卡片,永不逐条轰炸;过期问候直接丢弃(遥测留痕)。
- **共处模式**:菜单"陪我干活"→ 她安静做自己的事 + 久坐提醒 + 结束一句收尾;这是产品的日常主场景。

### 3.4 记忆层
- 身体事件(投喂/触摸日聚合/共处里程碑)写入 shared_moment,与语言记忆同库;检索自然回流到对话与主动消息。
- 长期弧:认识天数、习惯学习(她逐渐知道你几点来、爱聊什么)——只通过话语自然流露,无任何徽章/统计 UI。
- 记忆管理(查看/编辑/删除)只在 Web 机房。

### 3.5 关系层
- 引擎的关系模型是唯一真相源;身体的亲疏用**行为参数**表达:idle 时离光标的默认距离、表情基线、语气模板选择。
- 无可见数值(宪法第 6 条)。遥测记录关系轨迹供论文分析。

### 3.6 声音与渠道层
- 气泡文本为主,**≤2 句**;长内容说"来聊天窗说"并打开壳内聊天面板。
- TTS:终态含,默认关,设置里一键开。
- **一魂三身**:桌面壳 = 主身体;QQ = 口袋身体(同一灵魂同一记忆,外出时她在你口袋里,回家她在桌上,还记得路上聊的);Web = 机房(管理/评测,永不承载关系)。QQ 共魂在形态上定死,但工程槽位锁死为**论文数据分析完成后的第一个里程碑**;纪律:冻结日前 QQ 相关代码提交数必须为 0(可审计)。

## 4. 宿主与身体来源

- **主案 O2**:自建壳「小布壳」(WPF/Windows)+ 嵌 `VPet-Simulator.Core`(Apache-2.0)作动画/渲染器官;不引入游戏层(VPet-Simulator.Windows)。
- **美术**:沿用 VPet 默认宠物动画素材——版权归虚拟主播模拟器制作组,非商用免费,义务:署名 + 链接(写进关于页与论文致谢);若商业化需邮件授权。
- **spike 门(三出口,PM 轮修订)**:形态与宿主解耦依然成立(本文不随宿主改),但**路线必须有真实出口**:
  - 出口 a:O2 Core hello-world 在 ≤2 个 Windows 日内跑通 → O2 用 Core;
  - 出口 b:Core 失败但素材可自播 → **自写播放器成本重估后再决策**,不自动继续 O2;
  - 出口 c(放弃线):双 spike 累计超 4 个 Windows 日未过,或 M2 timebox 到期六拍未全过 → **O1 冻结开实验**,不恋战。
  - 顺序铁律:**O1 验证性一跳先于一切 O2 工作,单独占一个 Windows 日**——它是两条路线共享的后端地基验证,也是给"报废资产"定价的唯一方式。
- macOS 常驻(你的主力桌面)= 远景,需自有美术;形态届时不变,只换身体。

## 5. 完成线与流程

**双重完成线(PM 轮修订)**:功能线 = 六拍各真实发生一次 + 周检三项一轮;时间线 = §9 各里程碑 timebox。**两线取先到者**——timebox 到期未过功能线,自动触发出口 c(O1 冻结开实验),本条与"防翻案"决策同等效力。

- **验收客观化**:六拍逐拍写进 `VPET_ACCEPTANCE.md`(触发条件 + 预期遥测 SQL + 截图/录屏要求);证据包交 Codex 按清单审计,Codex 签"证据完备",不是作者签"我觉得发生了"。钟点锚定拍(09:12/00:30 等)配时钟模拟开关,真机钟点验收只留拍 5、6 各一次。
- **冻结机制**:git tag + config 哈希入库;实验期变更规则一页纸——仅 crash/丢数据可 hotfix,记 deviation log,当天标污染日。
- **实验设计重推导**:生理入脑后 spec §2 的 `body_state_injection` 开关语义已蒸发,ABAB 矩阵、适应期、有效日规则(在机 ≥2h 才计入)须按新架构重写并预注册,**不是照跑旧表**。
- **预算货币**:本项目第一预算单位是"Windows 小时"(每天 ≥2h),每个里程碑标注消耗;Codex 加速打字,不加速验证。
