# mini 分支 —— Agent 工作守则

> 本分支从 3935d27(v3 之前最后一个可用身体快照)分叉,目标:一个人
> 能读懂、讲明白、快速改结构的最小人格引擎。与 AGENTS.md 逐字节一致。

## 教训(为什么有这个分支)

v3 用 5 天长成 20 万行"测试全绿"的哑巴;之后的改造计划改了七版把
所有者改吐了。**文档不再接受评审——只有跑着的代码和她说出的话接受
评审。**测试全绿 ≠ 完成;计划完美 ≠ 开工。

## 品味(所有者裁决,不可协商)

- **纯陪伴**:她不做任务。天气/提醒/搜索/笔记/QQ 已删,不得回加。
- **她自己的生活是核心欲望**,优先级仅次于"接住对话"。
- **她可以关心地问**,但不催回、不追问、未回应零痕迹。
- **亲密浓度未定**:不在 prompt 或文档中加深、固化浓度措辞。
- **记忆全新开始**:不从 v3 世界迁移数据。

## 红线(四条,实现为 validators 里可读的纯函数,管全部写入不只管台词)

不索取 / 不编造 / 无总分 / 不撤回。等强度迁移主分支回归例。

## 尺寸与风格

- 引擎机器侧目标 ≤5000 行(数字仅所有者可改);人格素材/记忆/生活
  事实是数据,不计行数。
- JSON/JSONL 文件态、单进程、单写者;不建:数据库/DDL、事件溯源/
  重放/哈希权威、worker/supervisor、队列、向量库、依赖注入、版本化
  内部协议、多实例框架。
- 状态文件用临时文件替换写入;失败候选与拒因保存原文。
- 两个方案选小的;能删不加;没有第二个实现前不抽象接口。

## 完成的定义

改动 → 跑起来 → 她真实说出的话贴进总结 → 一句话回答"她哪里更活了"。

## 设计文档

`DESIGN.md` 是唯一设计文档(十几行),**永不改版评审**;要改就直接
改代码,DESIGN.md 事后如实记录。主仓 docs/(宪法、ENGINE_REBUILD_PLAN
v2.3、BODY_CONTRACT)只作只读素材。
