# 最小人格引擎 —— 设计(唯一文档,事后如实记录,不做版本评审)

三轮评审真正沉淀下来的十来行,其余边跑边定:

1. 一次回合:经历进来 → 按流逝时间推进状态/生活 → 选上下文 →
   一次模型调用出整包 {状态改动, 记忆操作, 表达?} → 红线校验整包
   → 一次提交 → 表达给身体。失败整包丢弃,只留诊断原文。
2. 红线管全部写入,不只管台词(未回应不得写成负面状态或记忆)。
3. 她的话只有真正显示(shown)后才进共同历史。
4. 直接回复永不静默:一次调用 → 一次带拒因重试 → 诚实的静态接住句。
   她自发的表达一律 ambient,可忽略、可静默。
5. 生活内容当场生成(推进时那次调用写出"读到什么想到什么"),
   不许回场补写;生活事件先在 history,有意义的才进长期记忆。
6. 记忆四类:用户事实/偏好、她的经历、共同经历、有证据的模式
   (模式须多例或用户确认);五动词 record/integrate/recall/correct/
   forget;temporary 不落盘。
7. 她的时钟由引擎自产;身体只报"看见了什么"。
8. 主动稀疏:在场才主动,每日至多一两次,频率永不随关系上调。
9. 数据 = state.json + history.jsonl + memories.json + failures.jsonl。
10. 持续基线:ambient 的身体提示保持到被替换;引擎离线回安全姿态。
