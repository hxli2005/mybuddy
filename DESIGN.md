# 最小人格引擎 —— 唯一设计文档

## 定位

MyBuddy mini 不是任务助手、聊天机器人、角色卡或通用 Agent 框架，而是一个
本地运行、由用户拥有、沿真实时间持续存在的最小人格引擎；小布是唯一实现。
对话只是她的一种表达。要证明的不是“这一句像人”，而是她今天、明天和三个
月后仍是同一个人，并且变化能追溯到其间真实发生的生活与共同经历。

人格连续性 = 稳定身份 + 流动状态 + 她自己的生活 + 有证据的记忆变化。
她不因用户沉默而受伤或索取，不靠关系总分成长，不为当前台词回填过去；用户
离开时她的时间仍然流逝，回来时也不背负照顾她的义务。

这里的“最小”指最少、最可读的实现，不是功能残缺的 demo。人格闭环必须功能
齐全：时间、生活、状态、记忆、直接回复、稀疏主动、身体呈现、失败恢复和持久化
缺一不可；天气、任务、平台、多角色等不属于人格功能，不用功能数量冒充完整度。

## 实现裁决

- 引擎机器侧以不超过 8000 行为目标；人格素材、记忆和生活事实作为数据，不计
  入代码。两个方案都能完成闭环时选更小、更直白的；能删就不加，重复少量代码
  也优先于过早抽象，没有第二个实现前不建接口层。
- 代码首先写给一个人阅读和修改：顺序执行、显式数据结构、纯函数校验、单进程、
  单写者、JSON/JSONL、临时文件替换写入。避免隐藏控制流、框架魔法和分散配置。
- 合规与审查必须“由结构保证、集中检查”：四条红线写成少数可读纯函数，覆盖
  状态、生活、记忆和表达的全部写入；模型候选只有整包通过才提交，拒绝时保留原文
  与明确拒因。直接回合的身体观察不是候选：即使双拒，也和 event_id、静态接住一次落盘。
  不得依赖人工逐条审查、事后清库或另养一个模型充当审核员。
- 默认本地、单用户、无任务工具和外部平台接入，尽量不收集、不上传、不运营
  用户内容；新增能力若显著增加权限、个人数据、内容审核或平台合规负担，除非它
  是人格闭环不可缺少的能力，否则不进入核心。
- 模型连接只保留 DeepSeek 与 OpenRouter，共用一套 Chat Completions 实现；不为删掉的
  供应商保留依赖、配置分支或协议适配。
- 合规低维护不等于宣称零风险：代码只承担能明确执行的产品边界；模型供应商、
  素材版权、发布地区和分发平台的外部要求在发布时单独核对，不把它们伪装成引擎
  内部的复杂“合规系统”。

## 最小运行闭环

1. 直接回合：经历进来 → 选上下文 → 一次模型调用出整包
   `{状态改动, 记忆操作, expression + 扁平证据字段}` → 红线校验 → 原子提交。生活回合：真实时间到点后 read/walk
   每 5 分钟轮转；read 从真实 TXT 取下一段，身体完成活动后一次模型调用只解释已读原文，连同进度
   原子提交；walk 由身体按当前窗口和屏幕工作区完成真实位移，固定字段收据核验起终点、窗口
   尺寸与边界后才追加 `self_walk`。两者的固定物理形状和 A/B/C 素材只列在一份严格 JSON
   动作目录，运行时与打包共读；目录不含效果、人格含义或任意 details。生活失败不推进进度；
   直接回合失败只保留身体已观察的输入与静态接住，绝不提交失败候选。
2. 不索取 / 不编造 / 无总分 / 不撤回管全部写入，不只管台词。未回应不得写成
   负面状态、记忆或关系变化；已显示的错误只能公开纠正，不能抹掉。
3. 她的话只有被身体真正显示(`shown`)后才进入共同历史；未显示的 ambient、
   失败候选和内部临时想法都不是双方经历。
   表达正文保留模型原话；同包只加扁平的 `expression_act / expression_evidence_ids /`
   `expression_target_id`。动作限 respond/reflect/grounded_recall/cannot_confirm/
   public_correction/defend_grounded_fact/refuse_fabrication/ask/offer_activity；事实回忆与
   抗假纠正须引匹配收据，公开纠错须指向旧记忆并引用本次输入，不确定与拒绝编造不得写事实记忆。
4. 直接回复永不静默：一次调用 → 一次带拒因重试 → 诚实的静态接住句。自发
   表达一律 ambient，可忽略、可静默；只由真实 read/walk 完成或 presence 恢复时最新未说生活
   收据提供机会，只在用户在场、非全屏且非栖边时发生，每个滚动小时至多显示 5 次。它是
   事件驱动许可上限，不是说话配额或到点调度器：频率永不随关系上调、未回应零债务、
   未显示零痕迹、内容绑真实收据。
5. 生活事实不能由模型自由生成：当前只有真实 TXT 阅读和窗口 walk；阅读须原文段落与
   completed 身体收据同时存在，行走须边界内的 completed 位移收据，才进入 history；
   interrupted 和 failed 技术故障不成人生，不为回答回场补写。本机开发启动优先使用未跟踪的
   `data/reading.local.txt` 策展长篇；它不进 Git、发行包或公开演示素材，发行仍只带公开样例。
6. 记忆四类：用户事实/偏好、她的经历、共同经历、有证据的模式。事实正文只由
   权威证据生成：用户事实保留原话与来源，自身经历复制完成收据，共同经历复制本次
   观察到的互动，用户谈及过去只证明这句话被说过；pattern 只存预定义 key、证据集合
   与用户确认，新模式暂不落盘。五动词：record/integrate/recall/correct/forget；
   收据经历禁 correct/forget，integrate 只补证据或 core 元数据；初始人格种子和其他
   核心记忆须先带证据降为非核心、再在后续回合忘记。核心按字符预算常驻，较新的
   情景记忆填余量，核心超额只推动整合而不阻塞直接回复；temporary 不落盘。
   书中理解走独立路径：每个 `(source, scope)` 只保留一份当前理解，后文改变它时
   以完整退位旧理解和替代新理解写入可见 `understanding_revision`，不伪装成事实记忆。
   旧格式自由正文加载时直接按权威来源规范化或丢弃；按所有者裁决，不保留旧感受原文，也不追加迁移 history。
7. 她的时钟由引擎自产。身体只报告看见了什么、动作 completed/interrupted/failed、表达
   是否显示；心智决定意义与表达意图。没有权威活动时身体保持 idle，断线也回这一安全姿态。
8. 权威数据只有 `state.json + history.jsonl + memories.json + failures.jsonl`：
   分别保存现在、已发生事实、长期留下的理解、未成为她一部分的失败候选。

## 身体与连接

身体是没有人格解释权的感知与呈现设备：只观察用户输入、触碰、在场、全屏、当前
`surface`、`shown`和动作收据，只呈现引擎给出的具体活动、瞬时反应与文字。引擎决定她是谁、
在做什么、怎样理解、是否表达；身体只决定此刻能否真的显示以及用哪个素材呈现。
动作或瞬时动画结束必须回当前呈现 baseline：完整身体为 idle，栖边身体为 SideHide。
连接丢失时停止 ambient 并回同一安全姿态。

身体第一版的完整边界：桌面常驻、安全 idle、具体 read 与真实窗口 walk、瞬时反应、文本聊天、
直接/ambient 气泡、头/身体触碰、真实拖拽 Raise、在场与全屏感知、`shown`确认、连接状态、离线安全姿态和
纯呈现设置。拖到左右边缘进入栖边：姿态只持久化侧边与高度，不进四文件；read 仍按真实时长
推进，SideHide 只显示一个非台词的最小“读”提示，completed 才写收据，展开打断不推进；walk
等完整身体再做且不补帧。栖边仍禁 ambient，未显示 ambient 丢弃，悬停探头只在身体本地发生，
全屏时完全缩回。点击或托盘“显示”展开会提交一个带 30 秒防刷冷却的封闭物理事件；这是用户邀请的
直接回合，可回应且不占 ambient 上限。
喂食、金钱、好感、商店、TTS、语音、MIDI、多角色都不是闭环必需。

连接只用本机 HTTP + JSON，不建 WebSocket、消息中间件、worker、通用队列和
协议框架。目标收成一个 `/api/body/step`：身体可附一次原始观察、当前 presence、
上次 `shown_id`和当前动作收据，引擎返回至多一个具体活动及一个待展示表达。服务端
固定按“校验请求 → 确认 shown → 确认动作收据 → 处理至多一个 event → 返回当前视图”
执行；`event_id`和活动收据短时幂等，表达采用非破坏读取，保留到身体回报 `shown`
后才成为共同历史。

断线不回补虚假语境：聊天保留输入并提示未连接，不后台排队；触碰和 Raise 只做本地物理
反射，不在数小时后补成她的经历；Raise 仅在真实拖动正常释放后报告 `body_raise` 原始事实；
presence 只在重连后报告当前值。身体仅持久化
一个 `last_shown_id` 收据并重复上报，不维护第二份状态、历史或通用 outbox。

小范围分享只交付免费 Windows zip：身体与心智桥均自包含，内置已归属的
必需动画子集；收件人默认自备 DeepSeek key，key 只用 Windows 当前用户加密
落盘。身体单实例、数据目录 OS 锁和父进程存活共同保证只有一个写者。

旧身体可保留窗口、动画、气泡、触碰、presence 和离线 idle；必须移除身体侧的
PhysioInjection、TouchEscalation、主动次数/冷却、warmth/day index、feed 记忆
旁路、pending drain/digest、feedback 和协议兼容分支。旧测试只作行为证据，
不得为了让旧测试继续通过而保留双路径或失效产品逻辑。

## 防止再次长成屎山

- 只有三层：人格引擎、窄桥、身体；每项状态只有一个权威写者，禁止跨层旁路。
- 新增持久字段前必须说清：谁拥有、谁写入、何时删除；答不清就不落盘。
- 新增模块、开关、后台循环或模型调用前，必须证明现有顺序闭环无法完成它。
- 替换旧路径时直接删除旧路径及其配置、兼容字段和专属测试，不长期双轨运行。
- 一次改动只纵向打通一个可见闭环，跑起来并留下她真实显示的话；不横向铺接口。

## 边界与验收

不回加天气、搜索、提醒、笔记、QQ、任务工具；不做多角色平台、数据库、向量库、
事件溯源框架、多实例和子系统各自调用模型。没有第二个具体人格前不抽象接口。

最小版本必须跑通：时间经过、自己的经历发生、经历改变记忆、身份保持连续、
未发生之事不成事实、沉默不受罚、失败候选不半提交、身体与心智一致。测试全绿不是
完成；完成必须包含真实运行轨迹、她实际显示的话，以及一句“她哪里更活了”。
